## Welcome Contributors!

Contributions are welcome! This guide will help you contribute effectively to the Bow Unity project.

## Getting Started

### Prerequisites

- Unity (version compatible with the project)
- Basic understanding of C# and Unity physics
- Git for version control

### Setting Up Your Development Environment

<Steps>
  <Step title="Fork the Repository">
    Fork the project repository to your GitHub account
  </Step>
  <Step title="Clone Your Fork">
    ```bash
    git clone https://github.com/YOUR_USERNAME/bow-unity.git
    ```
  </Step>
  <Step title="Open in Unity">
    Open the project in Unity and verify everything works
  </Step>
  <Step title="Create a Branch">
    ```bash
    git checkout -b feature/your-feature-name
    ```
  </Step>
</Steps>

## How to Submit Pull Requests

### Pull Request Process

<Steps>
  <Step title="Make Your Changes">
    Implement your feature or bug fix following the code style guidelines below
  </Step>
  <Step title="Test Thoroughly">
    Test your changes in Unity to ensure they work as expected and don't break existing functionality
  </Step>
  <Step title="Commit Your Changes">
    ```bash
    git add .
    git commit -m "Add: Brief description of your changes"
    ```
  </Step>
  <Step title="Push to Your Fork">
    ```bash
    git push origin feature/your-feature-name
    ```
  </Step>
  <Step title="Open a Pull Request">
    Go to the original repository and open a pull request from your branch
  </Step>
</Steps>

### Pull Request Guidelines

- **Clear Title**: Use a descriptive title that summarizes your changes
- **Description**: Explain what changes you made and why
- **Reference Issues**: If your PR addresses an issue, reference it (e.g., "Fixes #123")
- **Screenshots**: Include screenshots or GIFs if your changes affect visual elements
- **Testing**: Describe how you tested your changes

## Code Style Guidelines

To maintain consistency with the existing codebase, please follow these conventions:

### C# Naming Conventions

```csharp
// Public fields: camelCase
public float minStrength = 2f;
public GameObject projectilePrefab;

// Private fields: camelCase
private float percentage = 0;
private float currentDrawTime = 0;

// Methods: PascalCase
private void DrawBow()
private void StopDrawBow()
private void RenderLine()

// Properties: PascalCase
public Transform UpperLine { get; set; }
```

### Code Organization

```csharp
public class ExampleScript : MonoBehaviour
{
    // 1. Serialized fields with [Header] attributes
    [Header("Settings")]
    public float exampleValue = 1f;
    
    [Header("References")]
    public Transform exampleTransform;
    
    // 2. Private fields
    private float internalValue = 0;
    
    // 3. Unity lifecycle methods
    void Start() { }
    void Update() { }
    
    // 4. Custom methods
    private void CustomMethod() { }
}
```

### Comments and Documentation

- Use comments to explain **why**, not **what**
- Add headers to group related fields: `[Header("Bow Settings")]`
- Keep comments concise and meaningful

### Formatting

- **Indentation**: Use spaces (4 spaces per indent level)
- **Braces**: Opening brace on same line
- **Spacing**: Add blank lines between methods for readability

## Testing Your Changes

### Testing Checklist

<Steps>
  <Step title="Basic Functionality">
    - Bow draws when holding mouse button
    - Arrow fires when releasing mouse button
    - Arrow strength increases with draw time
    - String renders correctly
  </Step>
  <Step title="Edge Cases">
    - Rapid clicking (quick shots)
    - Holding for extended periods
    - Firing at different angles
    - Multiple arrows in the air
  </Step>
  <Step title="Performance">
    - No console errors or warnings
    - Smooth framerate maintained
    - Memory usage is stable
  </Step>
</Steps>

## Questions or Issues?

If you have questions or run into issues:

1. Check the existing issues on GitHub
2. Search for similar problems or questions
3. Open a new issue with detailed information:
   - Unity version
   - Steps to reproduce
   - Expected vs actual behavior
   - Screenshots or error messages

<Tip>
  When reporting bugs, include your Unity version and any console errors. This helps maintainers debug issues faster!
</Tip>
